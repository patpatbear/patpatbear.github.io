what gets(char* buffer) exactly does

when testing wheather gets(char* buffer append input to buffer, i used the following code: 

#include <stdio.h>
#include <math.h>

char buffer[100];
int d;

int main(){
  buffer = "hello";
  printf("%s\n",buffer);
  gets(buffer);
  printf("%s\n",buffer);
}

interestirngly, g++ output following message : 

g++ test.cpp
test.cpp: In function ‘int main()’:
test.cpp:8:12: error: incompatible types in assignment of ‘const char [6]’ to ‘char [100]’

it appears that the g++ treats buffer as char[100] and "hello" as const char [6], incase you wonder, even if buffer is char[6], it's an invalid assignment

conclusion 1: string literals can't assigned to string array.

changing the code to : 

char buffer[100];
int d;

int main(){
  strcpy(buffer,"hello");
  printf("%s\n",buffer);
  gets(buffer);
  printf("%s\n",buffer);
}

with input world, the output of the program is : 

hello
world

so, gets does not append to string, instead it writes directly to the buffer.

conclusion 2: gets does not append, memset before gets is not necessary. note: gets return NULL on failure, return str on success

why emacs breaks hard link

brief answer: emacs new up a new file when saved (the old-file was to renamed to old-file~)

hard link is somewhat alias to linked target (like c++ reference), while soft link is a file that stores the path of linked target (like c++ pointer).

workaround: use softlink (although file maybe newed up, the filename doesn't changed)
