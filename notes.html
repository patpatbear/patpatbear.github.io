<stdio.h> : input/output handling

  • I/O streams are objects of type FILE, each iostream is assosiated with an

external phisical device

  • I/O streams are locale sensitive, all streams access the same locale object, the one most recently installed with setlocale.
  • each I/O stream hold at least following information:
    • character width (unset, narrow, wide)
    • buffer state
    • the buffer
    • I/O mode
    • Binary/Text mode
    • EOF indicator
    • Error status indicator
    • file position indicator
    • reetrant lock used for concurrency
  • Binary/Text modes diffes in handling '\n' on windows, while POSIX does not distinguish those two modes.
  • narrow/wide orientation
    • newly opened stream has no orientation
    • the first io function called determine the orientation.
  • file access
    • fopen
    • freopen
    • fclose
    • fflush
    • setbuf

  • formated input/output
    • fscanf, scanf, sscanf
    • vscanf, vfscanf, vsscanf
    • printf, fprintf, sprintf, snprintf
      • format: %[-,+, ,#,0(modifiers)][*,integer(mininum-filed-width)][.integer,*(precision-of-conversion)][char*(length-modifier)]conversion-format-specifier
      • modifiers:-, left-justified; +, always add sign on; ' ', pad with space; #, alternative form; 0, padd with 0
      • minimum-field-width, specify the minumum width of the field; * would need an extra int to specify the filed
      • precision-of-conversion: meaning depends on conversion type
      • length-modifier: conversion type depended
      • conversion-format-specifier: s, c, g, a, f, x
